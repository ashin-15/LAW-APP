## 2024-05-14 - Expensive computations inside useMemo depending on search input
**Learning:** In React applications with large sets of structured data (like `ApprovedLaws`), performing complex string parsing, array grouping, deduplication, and mapping operations inside a `useMemo` that directly depends on `searchQuery` causes the expensive operation to block the main thread on every single keystroke. In `src/components/RepositoryView.tsx`, `organizeLawsBySource` (which builds chunks, hashes, groups, sorts) was called unconditionally on `searchQuery` change for all visible domains.
**Action:** When filtering complex data locally based on user input, split the work: create one `useMemo` to construct the heavy searchable index/groups that depends *only* on the base data array, and a second `useMemo` that performs the lightweight text filtering across the pre-computed groups.
